FEDERATRED LEARNING

What is Federated Learning?

Federated learning is a machine learning approach that allows a model to be trained across multiple decentralized edge devices (such as smartphones, IoT devices, or local servers) without exchanging raw data. The training process occurs locally on each device, and only model updates (gradients) are shared with a central server or aggregator.

Why Federated Learning?

Federated learning is necessary primarily due to privacy concerns. In conventional centralized machine learning, aggregating and analyzing data centrally raises significant privacy risks, as raw, sensitive data must be shared. Federated learning mitigates these risks by allowing model training on local devices without transmitting the raw data.

How it works?

STEP 1

The central server defines the problem, explores various statistical models, and selects one based on predefined criteria. The chosen model undergoes hyperparameter tuning, evaluation, and is deployed for training and inference. Continuous monitoring ensures ongoing performance and potential retraining as needed.

STEP 2

The central server distributes the chosen statistical model to multiple nodes. These nodes then engage in collaborative learning, each refining the model based on local data. The updated models are sent back to the central server for aggregation and further iteration.

STEP 3

The central server disseminates the initial model to individual nodes, where each node trains the model locally using its own dataset. After local training, the updated models are sent back to the central server for aggregation or ensemble to create a refined global model.

STEP 4

The central server aggregates the locally trained models' results without accessing the raw data, producing a consolidated global model. This privacy-preserving approach allows collaborative learning without compromising individual data privacy on the nodes.
